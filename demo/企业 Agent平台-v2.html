平台核心架构三大支柱

我们的平台构建在三大核心支柱之上，共同支撑起企业智能化的未来。

智能认知与决策引擎

作为平台的“大脑”，负责深度理解信息、管理知识、进行推理决策，是所有智能体实现高级智能的基石。

自然语言理解

使智能体能准确捕捉用户通过自然语言传递的显性需求与隐性情感，理解复杂、口语化的表述。

动态对话管理 (DM)

实现流畅、自然、多轮次的连贯对话，确保交互的逻辑性和用户体验的顺滑性。

LLM集成与赋能中心

充分、安全、高效地利用LLM的泛化能力，并通过Prompt工程和RAG框架将其融入特定业务场景。

企业知识中枢 (KMS)

构建和维护一个动态更新、可信赖的企业级“知识大脑”，为AI提供精准、高质量的上下文信息。

深度用户理解与个性化引擎

驱动智能体提供真正千人千面、精准触达、甚至超越用户预期的个性化服务与内容推荐。

智能体构建与流程编排中心

作为智能体的“设计工坊”，赋能业务人员通过低代码/无代码方式，直观地设计Agent行为逻辑、编排复杂业务流程。

低代码/无代码构建工作台

大幅降低智能体应用的构建门槛和周期，提供模板市场、多渠道部署和灰度发布能力，使业务创新能够快速落地验证。

智能流程自动化引擎

将重复性业务流程及多系统协作任务，通过可视化编排和事件驱动实现高效自动化，并支持新流程的优化与验证。

开放生态与运营管理平台

作为平台的“指挥塔”，负责为智能体提供运行所需的一切外部能力和保障，并对其全生命周期进行精细化监控与管理。

统一能力接入与适配网关

实现对企业内外部各类原子能力的“即插即用”、灵活替换和统一治理，为Agent构建一个开放、可扩展的能力生态。

一体化智能运营与分析平台

实现对智能体全生命周期的精细化管理、数据驱动的持续优化和人机协同的服务闭环。

平台基础与安全保障

提供坚实、安全、可靠的运行环境和集成基础，建立统一认证授权与安全凭证管理，保障数据安全交换。

平台能力演进四维框架

完整映射平台架构三大支柱，清晰指引平台演进路径。

自主性 - 能做什么

  • L1 - 流程化响应: 能引导用户完成预定义的、结构化的流程。
  • L2 - 主动式互动: 能基于事件或规则，主动发起交互、提供建议。
  • L3 - 协作式服务: 能自主规划、调用多工具、跨系统执行复杂任务，并与其他Agent协作。

智能决策 - 有多聪明

  • L1 - 逻辑推理: 理解上下文、进行简单的因果和逻辑判断，支持流程化对话。
  • L2 - 策略推荐: 结合业务策略模型和基础用户画像，提供个性化的、有策略依据的建议。
  • L3 - 目标驱动规划: 能基于模糊目标，自主规划并分解任务，动态调整执行路径。

使用成本 - 多好用

  • L1 - 模板化配置: 业务人员可在预设模板基础上，通过配置界面修改和创建助手。
  • L2 - 可视化编排: 业务人员可通过拖拽方式，可视化地编排对话流和简单业务流程。
  • L3 - 生态化共建: 平台提供开放市场，鼓励共同贡献、分享和复用Agent模板与组件。

生态连接 - 能连多少

  • L1 - 标准化接入: 提供标准化的连接器，能快速接入同类系统。
  • L2 - 平台化网关: 建立统一能力接入网关，支持连接更多企业内部系统。
  • L3 - 开放生态: 提供开放API/SDK，支持双向集成，形成内外联动的协作网络。

分阶段实施路线图

我们将分阶段推进这个项目，确保每一步的投入都有清晰的回报，最终实现宏伟蓝图。

现状：分析与问题识别

当前状态：现有 2 款游戏已接入独立的AI助手，均为“烟囱式”独立开发。能力局限于“被动式问答”，依赖第三方知识库，不具备主动沟通或工具调用能力。

核心痛点：

  • 重复建设，成本高昂：每接入新游戏或场景，都需从零开始开发。
  • 能力无法复用：成功的经验、AI能力、沉淀的知识无法共享。
  • 运营效率低下：知识库分散且依赖第三方，运营无法自主高效管理。
  • 服务模式单一：纯被动响应无法满足复杂运营需求，价值天花板低。

第一阶段：试点突破 (6-9个月)

聚焦“游戏运营助手”的平台化升级与能力增强

能力定位与演进

将手工作坊升级为标准化流水线，解决降本增效问题的同时，快速验证主动服务的业务价值。

自主性 (L1→L2): 核心发展L1-流程化，实现引导式多步对话；同步探索L2-主动式，基于玩家行为主动提示。

智能决策 (L1→L2): 核心发展L1-逻辑推理，夯实对话基础；同步探索L2-策略推荐，引入简单个性化逻辑。

使用成本 (L1): 核心发展L1-模板化配置，构建SaaS后台，让运营通过界面配置助手，解决重复建设痛点。

生态连接 (L2): 核心发展L2-平台化网关雏形，实现多租户（游戏）的标准化、低成本快速接入。

核心任务

  • 平台架构升级: 重构为模块化、可配置的SaaS服务，支持多游戏管理。
  • 功能能力拓展: 引入更复杂的语义理解和多轮对话管理。
  • 探索个性化服务: 基于玩家画像尝试推荐不同攻略或活动。

预期成果

  • 成功将现有助手迁移至新SaaS平台架构，服务1-2款核心游戏。
  • 新游戏接入配置时间缩短至数天内。
  • 多轮对话成功率达到70%以上，初步实现个性化互动。
  • 验证SaaS架构的可行性和复用价值，为下阶段奠定基础。

第二阶段：通用化赋能

从“专用工具”到“通用平台”

能力定位与演进

将已验证能力抽象并产品化，使平台从“游戏专用工具”升级为“企业通用平台”，赋能新业务部门。

自主性 (L2): 核心发展L2-主动式，将主动服务能力产品化、通用化，实现从“被动”到“主动”的关键跨越。

智能决策 (L2): 核心发展L2-策略推荐，建立统一用户画像与个性化引擎，驱动有商业价值的精准策略推荐。

使用成本 (L2): 核心发展L2-可视化编排，提供拖拽式工作流与对话流编排工具，赋能非技术人员。

生态连接 (L2): 核心发展L2-平台化网关，正式建成并扩展统一能力网关，打通核心内部系统。

核心任务

  • 能力抽象与通用化：提炼核心AI能力，支持多元业务场景（如HR、IT）。
  • 扩展连接能力：对接核心办公系统，为主动服务奠基。
  • 构建模板市场：创建客服、HR助手模板，降低新业务部门使用门槛。

预期成果

  • “企业级智能助手平台”1.0版上线。
  • 成功赋能2-3个新业务部门（含游戏）。
  • 支持他们自主配置和管理不同类型的智能助手。

第三阶段：生态化运营

从“中央平台”到“赋能生态”

能力定位与演进

推动平台成为企业AI能力的核心中枢与创新孵化器，让智能化深入企业运营的毛细血管。

自主性 (L3): 核心发展L3-协作式，Agent可自主规划、调用工具，执行跨系统复杂任务，成为数字员工。

智能决策 (L3): 核心发展L3-目标驱动规划，基于融合数据与深度画像，理解模糊目标并自主规划行动策略。

使用成本 (L3): 核心发展L3-生态化共建，建立能力与模板市场，赋能“人人皆是Agent构建师”。

生态连接 (L3): 核心发展L3-开放生态，提供成熟API/SDK支持双向调用，形成内外联动的智能协作网络。

核心任务

  • 全面推广应用：在HR、IT、财务、销售等更多部门落地。
  • 建立模板与组件市场：促进能力复用，降低使用门槛。
  • 持续运营迭代：赋能业务人员成为“Agent构建师”。

预期成果

  • “人人皆可构建，处处皆有助手”，智能化成为公司日常运营的基石。
  • 核心指标达成或超过项目总体目标。
  • 平台成为企业AI能力和数据资产的核心中心，构筑长期竞争优势。

首个MVP版本详细计划 (3个月)

冲锋的号角已经吹响，这是我们迈向成功的第一步。

MVP版本说明

本次MVP其核心价值在于：验证核心技术架构、以及建立内外部信心。一切设计都将围绕“用最小的代价，跑通一个从‘助手配置’到‘玩家服务’再到‘数据反馈’的完整业务闭环”这一原则展开。

MVP 三大核心目标

平台可见

上线基础SaaS平台与运营工作台，支持运营人员无代码配置助手。

服务可见

实现核心“被动响应”场景，有效处理2-3个高频运营问题。

关怀可见

验证关键“主动关怀”场景，如基于事件自动发送祝贺消息。

MVP 核心交付物

聚焦于平台与AI能力的双重验证

SaaS平台与运营工作台建设 (平台雏形 v0.1)

技术实现逻辑: 搭建一个可用的SaaS平台骨架，核心是服务于单个助手实例的创建、配置和监控，为后续所有平台化能力奠定基础。

  • 基础数据看板：一个简洁的仪表盘，展示助手调用量、知识库命中率等核心指标。
  • 知识库管理界面：一个表格页面，运营人员可以在此对FAQ知识进行增、删、改、查，并支持批量导入。
  • 对话脚本编辑器：一个带有语法高亮的文本编辑区域，用于运营或技术人员粘贴和管理YAML/JSON格式的对话流脚本。
  • 渠道部署页面：显示为不同渠道（如游戏内H5、企业微信）生成的部署代码片段或配置信息。
  • ......

    • 底层AI基础能力建设 (AI平台雏形 v0.1)

    Agent能力目标: 聚焦被动响应 (RAG) 和简单主动关怀 (事件驱动)。

    NLU服务 (v0.1 - 场景聚焦)

    技术实现: 基于 Langchain 框架开发意图识别和实体提取能力，封装成初步服务。

    关键路径: 明确场景 → LLM提示词构建 → 提供简单API。

    DM服务 (v0.1 - 脚本驱动)

    技术实现: 构建对话脚本解释器（支持JSON/YAML），根据NLU结果加载并执行对应脚本。

    关键路径: 定义脚本格式 → 开发脚本解释器 → 实现状态管理。

    LLM集成服务 (v0.1 - RAG核心)

    技术实现: 重点实现RAG框架，负责问题向量化、知识检索、Prompt组合、调用LLM API。

    关键路径: LLM API选型与接入 → RAG流程实现 → Prompt设计。

    知识管理服务 (KMS v0.1)

    技术实现: 后端提供知识的增删改查接口，支持运营人员在后台上传和管理知识。接入火山/dify知识库/自己的向量数据库（适配层快速切换）。提供高效的语义检索API供LLM集成服务调用。

    关键路径: 选型向量数据库 -> 实现知识导入与向量化 -> 开发检索API。

    事件订阅与简单规则引擎 (v0.1)

    技术实现: 提供一个API接口，供游戏后端推送特定业务事件（如玩家升级）。内置一个轻量级规则匹配逻辑，根据运营后台配置的“事件-动作”规则，触发DM执行相应的主动消息发送脚本。

    关键路径: 定义事件API -> 开发规则匹配逻辑 -> 与DM联动。

    MVP后各阶段AI能力实现路径

    在MVP成功的基础上，清晰规划后续每个阶段AI能力的建设重点，确保技术与业务同步演进。

    第一阶段 (AI平台基础 v1.0)

    Agent能力目标: 支持更复杂的引导式流程对话和初步个性化推荐，并为多款游戏快速部署助手。

    SaaS平台与运营工作台演进

    技术实现: 支持多游戏/多助手实例管理，实现配置隔离与共享，并引入初步的权限管理。

    NLU服务 (v1.0 - 模型优化与服务化)

    技术实现: 提升意图识别和实体提取的准确率和泛化能力。将NLU彻底封装为独立的、可配置的微服务，供不同游戏实例共享和调用。

    DM服务 (v1.0 - 复杂逻辑支持)

    技术实现: 增强对话脚本解释器，支持条件判断、循环、变量传递和“调用子脚本/子流程”机制，实现复杂分支对话。

    LLM集成服务 (v1.0 - Prompt工程与管理)

    技术实现: 建立初步的Prompt模板库和管理机制，针对不同场景优化Prompt。探索更精细化的RAG策略（如多路召回、答案重排）。

    用户理解与个性化引擎 (雏形 v0.1)

    技术实现: 收集用户行为数据和关键标签，基于简单用户分群和规则引擎实现个性化内容触达。

    第二阶段 (企业级智能助手平台 v1.0)

    Agent能力目标: 赋能新业务部门（如HR/IT），实现基于事件的智能主动服务，并提供Agent模板。

    DM服务 (v2.0 - 可视化与策略化)

    技术实现: 探索提供简化版可视化对话流设计工具，并引入业务策略模型。DM能动态判断并调用子对话或触发Workflow来补全信息。

    企业知识中枢 (KMS v2.0 - 知识融合)

    技术实现: 支持接入更广泛的企业数据源（如钉钉文档），开始构建基础的企业知识图谱。

    智能流程自动化引擎 (雏形 v1.0)

    技术实现: 增强事件总线能力，订阅企业系统事件。支持通过简单配置，串联AI能力和API调用，实现初步的流程自动化。
